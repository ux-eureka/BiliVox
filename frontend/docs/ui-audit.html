BiliVox UI 审计与体验优化报告（本地版）

目标地址：http://localhost:3000/ 生成方式：代码审计 + 本地运行核对

0. 结论摘要（可执行）

  • 样式链路不闭环 会导致“改了 class 但页面不变”；需要补齐 PostCSS/Tailwind 配置。
  • 多套组件库并存（TDesign + Tailwind + Vuetify）导致断点、间距、圆角、阴影与交互反馈不一致。
  • 页面级布局缺少统一规则：容器宽度、padding 与固定侧栏宽度在多个地方重复定义，易漂移。

1. 现状 UI 设计语言盘点

1.1 组件体系

  • TDesign：主要页面布局与基础组件
  • Tailwind：大量 utility class 控制间距/排版/颜色
  • Vuetify：配置中心页面大量使用（体验割裂来源之一）

1.2 色彩 / 排版 / 间距

  • 颜色来源混用：TDesign token、Tailwind class、硬编码色值并存
  • 排版缺少字号阶梯：同层级组件字号/字重不一致
  • 断点与栅格三套并存：TDesign / Tailwind / Vuetify

2. 响应式与可访问性

响应式高风险点

  • 固定宽侧栏在平板宽度下易挤压主内容，建议仅在 xl 以上固定。
  • 同页 Grid + Row/Col 混搭容易出现断点错位，建议页面级统一 Grid。

可访问性要点（WCAG 2.1 AA）

  • 小字灰度需保证对比度（正文 ≥ 4.5:1）。
  • icon-only 按钮需要 aria-label/title。

3. 优先级问题清单

P0（高）

  • 样式链路不闭环导致“UI 不变化”（补齐 Tailwind/PostCSS）。
  • 多组件库并存导致一致性断裂（短期 token 统一，中长期统一组件体系）。

P1（中）

  • 层级不清：容器/卡片/卡片组缺少一致的 Surface 规则。
  • 响应式策略混乱：统一 Grid + 固定宽仅在大屏启用。

4. 已实施的立即修复（摘要）

  • 补齐 Tailwind/PostCSS 配置闭环，确保 utility class 可稳定生效。
  • 引入 tokens.css，统一颜色/圆角/阴影的“单一来源”。
  • 统计区域改为“大容器 + 三张小白卡”，提高层级可读性。
